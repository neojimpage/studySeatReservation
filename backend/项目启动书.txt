校园自习座位预约系统项目启动书
文档版本：V1.0编写日期：2026 年 3 月 9 日适用场景：软件工程实训项目立项交付开发技术栈：Java 17+ + Spring Boot 3.2 + MySQL 8.0 + Vue 3 + Element Plus
一、项目名称
校园自习座位预约系统
二、项目背景与意义
（一）项目背景
高校图书馆、专属自习室是学生日常学习的核心场景，目前普遍存在座位占座混乱、找座耗时、预约流程不透明等痛点；传统线下登记、微信群预约的方式，无法实现座位状态实时更新、预约流程规范化，既影响学生的学习体验，也难以形成有序的自习秩序。
本项目针对校园自习座位预约的真实痛点，基于MVP 最小可行性产品原则开发，聚焦学生端核心预约流程，摒弃管理端复杂功能，确保在 6 周实训周期内完成可运行的轻量化系统。
（二）项目意义
学生端：实现自习座位的线上自助预约、状态实时查看，解决占座、找座难问题，提升学习场景的便捷性和体验感;贴合软件工程实训要求，实现前后端分离开发、团队协作分工、全流程项目落地，让团队成员在实际开发中掌握需求分析、架构设计、编码实现、测试验收的完整工程流程。

四、核心功能模块列表（Backlog）
本项目聚焦学生端核心流程，所有功能均为 Must-have，无管理端功能，确保 6 周内完成 MVP 交付。
模块 1：用户登录与注册
US1-01：学生可通过学号 + 密码注册账号，初始密码为学号后 6 位
US1-02：学生可通过学号 + 密码登录系统，登录状态保持 7 天
US1-03：支持修改登录密码，退出登录功能
模块 2：座位状态展示
US2-01：以可视化列表 / 平面图展示座位，标注状态（空闲 / 已预约 / 占用 / 禁用）
US2-02：支持按座位号、区域筛选座位
US2-03：座位状态随预约、取消、暂离操作实时更新（延迟≤5 秒）
模块 3：学生预约管理
US3-01：学生可选择未来 7 天内的空闲座位，预约 2 小时时段，每日最多 2 个时段
US3-02：预约成功后生成有效记录，座位状态同步为 “已预约”
US3-03：预约开始前 30 分钟可取消预约，超时不可取消，记为爽约
US3-04：已到岗学生可提交暂离报备，系统保留座位 30 分钟，超时自动释放
US3-05：展示历史 / 当前预约记录，支持按时间倒序查看
模块 4：违规提醒与限制
US4-01：爽约累计 3 次，自动限制次日预约权限，次日 24 点后解除
US4-02：暂离超时自动释放座位，不计入爽约次数
五、项目边界与约束
功能边界：本版本不实现管理端功能、扫码签到、人脸识别、支付、座位推荐等非核心功能，仅聚焦学生端核心预约流程。
技术约束：严格使用 Java 17+、Spring Boot 3.2、MySQL 8.0、Vue 3 技术栈，遵循阿里巴巴 Java 开发手册。
时间约束：6 周实训周期内完成 MVP 开发与交付，每周提交可运行版本与进度报告。
